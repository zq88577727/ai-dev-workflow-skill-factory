# Failure Playbook

## 常见失败

### 直接写代码

症状：没有需求文档和技术方案就改文件。

修正：回到 S3/S4，生成并确认文档。

### 架构文档空泛

症状：只有目录列表，没有模块职责和调用链。

修正：补读入口、核心模块和测试，重写 `ARCHITECTURE.md`。

### AI 乱修 Bug

症状：没有假设和验证步骤，直接批量改动。

修正：回到 S7，列嫌疑文件、假设和最小验证。

### Review 失效

症状：实现和审查由同一视角完成，或只检查语法不检查逻辑。

修正：拆成逻辑 Review 和语法/编译/测试 Review。

### 验收失败仍交付

症状：缺文件、缺 review、缺 workflow-run，却说完成。

修正：运行验收脚本，按失败项补齐。
