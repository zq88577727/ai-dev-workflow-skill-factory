# Roadmap

## Near Term

- Keep `workflow-run.schema.json` as the canonical run contract.
- Add more expected-failure fixtures when new validation rules are added.
- Improve initialization so teams can create project-specific workflow folders
  without copying files by hand.

## Medium Term

- Add example workflows for frontend, backend and legacy-code maintenance.
- Add lightweight, standard and strict operating modes.
- Add preset rule packs for common project types.

## Later

- Publish a versioned release package.
- Add richer reporting around validation failures.
- Explore adapters for more AI coding tools without making the core workflow
  depend on a single vendor.
