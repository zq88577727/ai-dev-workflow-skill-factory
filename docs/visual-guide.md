# Visual Guide

这个文档说明 README 中 4 张图分别承担的表达任务。

## 1. Flowchart

![Workflow flowchart](../assets/01-workflow-flowchart.png)

用途：解释完整状态机。

它表达：

- 项目想法不是直接进入代码。
- 先经过架构、规则、需求和技术方案。
- harness 执行、双 Review 和测试是交付闸门。
- 最后沉淀为可复用 skill。

## 2. Example Workspace

![Example workspace](../assets/02-example-workspace.png)

用途：让读者理解这个项目落地后会看到什么。

它表达：

- 工作流运行后会留下文档产物。
- `Validate OK` 是交付前的检查点。
- 项目强调可审阅、可复盘、可继续改造。

## 3. Personified AI Teammate

![Personified AI teammate](../assets/03-personified-ai-teammate.png)

用途：把抽象的“AI 受控执行”变得直观。

它表达：

- AI 是工程队友，不是黑箱。
- 文档包、代码包和 review gate 逐步流转。
- 受控流程比自由发挥更稳定。

## 4. Hand-Drawn Skill Factory

![Hand-drawn skill factory](../assets/04-handdrawn-skill-factory.png)

用途：表达“从自己的想法生成工作流 skill”。

它表达：

- 用户的 idea 进入文档漏斗。
- 工作流把隐性经验变成显性产物。
- 最终输出 reusable AI skill。
- 每次运行都通过 learn / improve / reuse 继续迭代。
