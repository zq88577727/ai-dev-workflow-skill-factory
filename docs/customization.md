# Customization Guide

## For A Backend Service

- Add API contract rules to `PROJECT_RULES.md`.
- Add database migration policy.
- Require integration tests for changed endpoints.

## For A Frontend App

- Add component, state, accessibility and responsive layout rules.
- Require screenshot or browser smoke checks for UI changes.
- Keep design-system constraints explicit.

## For A Legacy C++ Project

- Make `ARCHITECTURE.md` deeper before changing code.
- Add build matrix and compiler constraints.
- Prefer small patches and targeted regression tests.

## For A Team Workflow

- Add PR template.
- Add commit message examples.
- Add ownership boundaries.
- Add release note requirements.
